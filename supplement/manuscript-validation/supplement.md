# Supplementary Material S1

## Reproducible fixture, competency queries, and entailment checks for GDPO v4.0.3

This supplement accompanies *Principles Are Not Properties: A BFO-Grounded Ontology for Normative Design Guidance and Evaluation Traceability*. It provides the article-specific fixture and executable checks supporting the Evaluation section. The repository’s baseline validation and this supplement serve different purposes: the baseline checks release structure and selected modeling constraints, whereas S1 reproduces the manuscript’s competency-query and OWL-RL results.

The supplement uses the current GDPO target-expression strategy. Principle categories are constrained through `owl:allValuesFrom` class-expression fillers on `aims at artifact-side target` (`GDPO0000454`). S1 contains no `owl:hasValue` target restriction and does not represent target universals by declaring GDPO classes as named individuals.

The product names and numerical scores in the fixture are illustrative test data. They are not empirical judgments about the named products.

## S1.1 Files and execution environment

The version-controlled package contains:

- `fixture.ttl`: the SK4/iPhone example ABox;
- `queries/`: six exercised competency-query forms and one additional comparison template;
- `run_validation.py`: deterministic structural, query, and OWL-RL checks;
- `requirements.txt`: RDFLib 7.2.1 and OWL-RL 7.1.4;
- `expected-output.txt`: the expected report used as a regression oracle; and
- `validation-output.txt`: the report regenerated by the runner.

Run the package from its directory:

```bash
python3 -m pip install -r requirements.txt
python3 run_validation.py
```

The runner exits with a failure if a query result count changes, a reported entailment disappears, `owl:hasValue` returns to the ontology, a GDPO class is also declared as a named individual, or the generated report differs from the expected output.

## S1.2 Test fixture

The fixture represents two illustrative material artifacts, method specifications, score components and scales, temporal regions, evaluation records, communication content, evaluation processes, an evaluator, and optional realization evidence. Its two principal test records are:

- `Eval_SK4_Durability_2024`, explicitly against the canonical durability statement and using `DurabilityRubric_v1`, which operationalizes both durability and environmental friendliness; and
- `Eval_SK4_ManualHonesty_2024`, explicitly against the canonical honesty statement and about `SK4_UserManual_v1`.

The full Turtle serialization is supplied as `fixture.ttl`. The generic type asserted for the manual-honesty record is design evaluation record (`GDPO0000044`). Design honesty evaluation (`GDPO0000455`) and design communicative honesty evaluation (`GDPO0000452`) are expected consequences rather than manually asserted shortcuts.

## S1.3 Expanded honesty target expression

The following Manchester-style rendering expands the target-side pattern used for the honesty worked example. Labels are shown for readability; the ontology uses the corresponding GDPO, BFO, and CCO IRIs.

```text
Class: PrincipleOfHonesty

SubClassOf:
  DesignPrinciple,
  aims at artifact-side target only
    (
      (
        DesignMaterialHonesty
        and inheres in some MaterialEntity
        and has basis in some DesignMaterialHonestyBasis
        and realized in only
          (DesignManufacturingProcess or DesignUseAndMaintenanceProcess)
      )
      or
      (
        DesignFunctionalHonesty
        and inheres in some MaterialEntity
        and has basis in some DesignFunctionalHonestyBasis
        and realized in only DesignInteractionProcess
      )
    )
```

This is a TBox constraint on the kinds of target the principle concerns. It does not assert that any particular artifact bears either disposition and does not establish satisfaction of the principle.

The six forms counted in the paper are A2.1, A2.2, A2.4, A2.5, A2.6, and A2.7: four instance-data queries and two schema-inspection queries. A2.3 is retained and tested as an additional cross-artifact comparison template.

Some schema-query bindings are anonymous OWL class-expression nodes. In particular, lifecycle qualifiers with union-valued fillers require an RDF-list traversal or a reporting layer to display each named union member. This is the tooling boundary described in the paper, not a failed query.

## S1.5 Entailment checks

OWL-RL materialization produces the two consequences reported in the manuscript:

1. `Eval_SK4_Durability_2024` has both `GDPO0000072` (canonical durability statement) and `GDPO0000074` (canonical environmental-friendliness statement) as evaluation-relevant principles. Durability remains the explicitly selected criterion; environmental friendliness is brought into scope by the method.
2. `Eval_SK4_ManualHonesty_2024` is classified as `GDPO0000044` (design evaluation record), `GDPO0000455` (design honesty evaluation), and `GDPO0000452` (design communicative honesty evaluation).

Neither result establishes that an artifact is durable, environmentally friendly, or honest. The tests verify the ontology’s traceability and classification behavior. Substantive product judgments require populated observations, evidence, justified methods, and interpretation of the relevant scale.

## S1.6 Scope of validation

The package validates parsing, the absence of the superseded has-value/punning target pattern, deterministic query results, method-derived relevance, and communicative-honesty classification under the pinned OWL-RL implementation. It does not constitute a fresh full OWL 2 DL consistency proof, an empirical product evaluation, or a complete validation theory for all ten Rams principles.
